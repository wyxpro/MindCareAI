# 灵愈AI数字医生需求文档

## 1. 应用概述

### 1.1 应用名称
灵愈AI数字医生

### 1.2 应用描述
灵愈AI是一个基于多模态AI的数字医生抑郁诊断与疗愈助手Web应用（适配移动端）。系统通过整合智能手环生理数据、多模态情绪识别（文本、语音、图片、面部表情）、主动式AI对话评估、以及个性化疗愈方案,为用户提供全方位的心理健康管理服务。

## 2. 核心功能

### 2.1 首页功能
- 健康状态概览展示
- 情绪趋势显示
- 待办提醒
- 智能手环数据展示
- 页面路由路径: /

### 2.2 记录页面功能
- 情绪日记记录
- 日历视图
- 5级量表打卡
- AI情绪分析
- 随手记功能:
  - 信息输入方式支持:
    - 文本输入:提供便捷的文本编辑框
    - 语音输入:支持语音识别并转换为文字
    - 图片上传:允许从相册选择或直接拍摄照片上传
  - 存储与关联功能:
    - 自动将每条记录保存到系统日历的对应日期中
    - 在日历日期上显示有记录的视觉标记
  - 信息查看交互:
    - 点击日历中的标记时,弹出卡片式弹窗
    - 弹窗内完整显示该日期的所有记录内容（文字、图片等）
    - 支持在弹窗内进行简单的编辑操作
  - 用户体验要求:
    - 输入界面简洁直观,操作流程顺畅
    - 弹窗动画过渡自然,关闭便捷
    - 不同输入方式之间有清晰的切换指引
- 页面路由路径: /record

### 2.3 评估页面功能
- 主动式AI对话评估
- 多模态情绪识别(文本、图片、语音、面部表情)
- 可视化评估报告
- 页面路由路径: /assessment

### 2.4 疗愈页面功能
- 冥想音频
- 睡眠辅助
- 专注训练
- 匿名树洞互助
- 页面路由路径: /healing

### 2.5 我的页面功能
- 个人资料管理
- 健康档案查看
- 手环连接管理
- 医生对接
- 页面路由路径: /profile

### 2.6 医生管理后台功能
- 登录与权限管理
- 可视化数据看板
- 知识库管理
- 患者管理
- 预警消息处理
- 页面路由前缀: /doctor

## 3. 技术要求

### 3.1 技术架构
- 前后端分离架构
- 适配移动端网页
- 明确定义API调用接口
- 定义数据格式
- 完整的页面路由结构
- 模块化代码组织

### 3.2 用户体验
- 现代化UI设计风格
- 协调的配色方案
- 视觉层次清晰
- 图标和视觉元素风格统一
- 用户交互动效良好
- 符合主流审美标准

## 4. 数据与安全

### 4.1 数据管理
- 数据持久化与同步
- 云端数据存储
- 实时数据同步
- 离线数据缓存

### 4.2 隐私与安全
- 用户敏感信息加密存储
- 身份验证机制
- 加密通道数据传输
- 数据删除机制
- 匿名树洞身份隔离
- 访问日志记录
